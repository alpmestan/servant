Union of two APIs, first takes precedence in case of overlap.

Example:

type MyApi = "books" :> Get [Book] -- GET /books
        :<|> "books" :> ReqBody Book :> Post Book -- POST /books

The generated docs for a :<|> b just append the docs for a with the docs for b.

A client querying function for a :<|> b will actually hand you one function for querying a and another one for querying b, stitching them together with :<|>, which really is just like a pair.

type MyApi = "books" :> Get [Book] -- GET /books
        :<|> "books" :> ReqBody Book :> Post Book -- POST /books

myApi :: Proxy MyApi

getAllBooks :: BaseUrl -> EitherT String IO [Book]

(getAllBooks :<|> postNewBook) = client myApi
(HasServer a, HasServer b) => HasServer ((:<|>) a b)

A server for a :<|> b first tries to match the request again the route represented by a and if it fails tries b. You must provide a request handler for each route.
